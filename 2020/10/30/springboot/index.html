2、多环境配置

  • 四种环境：开发dev、测试test、预生产pre、生产pro

  • 配置

    • spring boot允许通过命名约定按照一定的格式(application-{profile}.properties)来定义多个配置文件
    • 例如，在resources中创建application-dev.properties、application-test.properties、application-pro.properties，在application.properties中输入spring.profiles.active=dev，指定读取哪个配置文件。

3、打包上传

  • 去除相关生产环境没用的jar

  • 本地maven打包成jar包

    • mvn clean package -Dmaven.test.skip=true 跳过测试

  • 服务器安装jdk，上传Jar包

  • 启动jar包

    • java -jar xxxx.jar
    • 守护进程、系统服务、shell脚本
      • nohup java -jar xxxx.jar &

  • liunx 查看端口被哪个应用占用 lsof -i:port

  • 打包指定配置文件

    • 使用maven的profiles
    • 使用springboot的profile=active

  • 注意：

    • 阿里云防火墙是否开启，可以选择关闭，关闭是不安全的，可以选择开放端口
    • 阿里云的安全访问组，开启对应的端口，如果应用是以80端口启动，则默认可以访问

  • 本地提交生产代码->gitlab仓库->Jenkins自动化构建->运维或者开发人员发布

4、热部署

  • pom.xml添加依赖
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-devtools</artifactId>
            <optional>true</optional>
        </dependency>
<build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
                <configuration>
                    <!--必须添加这个配置-->
                    <fork>true</fork>
                </configuration>
            </plugin>
        </plugins>
    </build>

  • 注意

    • eclipse添加依赖即可
    • IDEA还需要配置

  • IDEA配置

    • 修改setting
    • 使用快捷键打开，选择Registry
      • 默认快捷键：
        • window快捷键 Shift+Ctrl+Alt+/
        • mac快捷键 Shift+Command+Alt+/
  • 重启IDEA

5、配置文件

  • 常见配置文件格式
    • xml、properties、json、yaml
  • Springboot里面常用xx.yml
    • YAML（Yet Another Markup Language）
    • 写 YAML 要比写 XML 快得多(无需关注标签或引号) 使用空格 Space 缩进表示分层，不同层次之间的缩进可以使用不同的空格数目
    • 注意：key后面的冒号，后面一定要跟一个空格,树状结构
server:
  port: 8080
  jetty:
    accesslog:
      append: true
      enabled: true
spring:
  redis:
    timeout: 1000
    port: 1212
    database: 0
  • Springboot里面常用 xx.properties
    • Key=Value格式
    • 语法简单，不容易出错
server.port=8082
#静态资源访问路径
spring.resources.static-locations = classpath:/META-INF/resources/,classpath:/resources/,classpath:/static/,classpath:/public/,classpath:/templates/
#session失效时间，30m表示30分钟
server.servlet.session.timeout=30m
# Maximum number of connections that the server accepts and processes at any given time.
server.tomcat.max-connections=10000
# Maximum size of the HTTP post content.
server.tomcat.max-http-post-size=2MB
server.tomcat.max-http-form-post-size=2MB
# Maximum amount of worker threads
server.tomcat.max-threads=200
  • 官方文档：https://docs.spring.io/spring-boot/docs/2.4.0/reference/htmlsingle/#core-properties

6、配置文件映射属性和实体类

  • 配置文件加载

    • 方式一
      • Controller上面配置 @PropertySource({“classpath:resource.properties”})
      • 增加属性 @Value(“${test.name}”) private String name;
    • 方式二
      • 实体类配置文件
        • 添加 @Component 注解；
        • 使用 @PropertySource 注解指定配置文件位置；
        • 使用 @ConfigurationProperties 注解，设置相关属性；（制定模块，可忽略）
        • 必须 通过注入IOC对象Resource 进来 ，才能在类中使用获取的配置文件值。
          @Autowired private ServerSettings serverSettings;

  • 测试pay.properties

#微信支付
wxpay.appid=a56219556165
#支付秘钥
wxpay.sercret=asdasdskcssa
#微信支付商户号
wxpay.mechid=5635656
  • 测试testController.java
@RestController
@RequestMapping("api/v1/test")
@PropertySource({"classpath:pay.properties"})
public class TestController {

    @Value("${wxpay.appid}")
    private String payAppid;

    @Value("${wxpay.sercret}")
    private String paySecret;

    /**
     * 注入wxConfig  方式二
     */
    @Autowired
    private WXConfig wxConfig;


    /**
     * 方式一
     */
    @GetMapping("test_config")
    public JsonData testConfig(){
        Map<String,String> map = new HashMap<>();
        map.put("appid",payAppid);
        map.put("secret",paySecret);
        return JsonData.buildSuccess(map);
    }
    /**
     *    方式二
     */
    @GetMapping("test_config_class")
    public JsonData testConfigClass(){
        Map<String,String> map = new HashMap<>();
        map.put("appid",wxConfig.getPayAppid());
        map.put("secret", wxConfig.getPaySecret());
        map.put("mechid",wxConfig.getPayMechId());
        return JsonData.buildSuccess(map);
    }
}
  • 测试WXConfig.java
@Configuration
@PropertySource(value = "classpath:pay.properties")
public class WXConfig {

    @Value("${wxpay.appid}")
    private String payAppid;

    @Value("${wxpay.sercret}")
    private String paySecret;

    @Value("${wxpay.mechid}")
    private String payMechId;

    public String getPayAppid() {
        return payAppid;
    }

    public void setPayAppid(String payAppid) {
        this.payAppid = payAppid;
    }

    public String getPaySecret() {
        return paySecret;
    }

    public void setPaySecret(String paySecret) {
        this.paySecret = paySecret;
    }

    public String getPayMechId() {
        return payMechId;
    }

    public void setPayMechId(String payMechId) {
        this.payMechId = payMechId;
    }
}
