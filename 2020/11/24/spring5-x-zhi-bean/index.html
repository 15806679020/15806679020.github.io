一、scope作用域

  • singleton：单例, 默认值，调用getBean方法返回是同一个对象,实例会被缓存起来，效率比较高，当一个bean被标识为singleton时候，spring的IOC容器中只会存在一个该bean

  • prototype: 多例，调用getBean方法创建不同的对象，会频繁的创建和销毁对象造成很大的开销

  • 其他用 （作用域 只在 WebApplicationContext）

    • request ：每个Http请求都会创建一个新的bean
    • session: 每个Http Session请求都会创建一个新的bean
    • global session（基本不用）
       <!--单例-->
    <bean name="video" class="club.ldclass.spring.domain.Video" scope="singleton">
        <property name="name" value="tom"/>
        <property name="id" value="23"/>
    </bean>

       <!--多例-->
    <bean name="video" class="club.ldclass.spring.domain.Video" scope="prototype">
        <property name="name" value="tom"/>
        <property name="id" value="23"/>
    </bean>
    private static void testScope(ApplicationContext applicationContext) {
        Video video = (Video) applicationContext.getBean("video");
        Video video1 = (Video) applicationContext.getBean("video");
        //scope为singleton时，控制台输出true；scope为prototype时，控制台输出为false
        System.out.println(video == video1);
    }

二、常见的注入方式

  • 通过实体类中set方法注入
    <bean name="video" class="club.ldclass.spring.domain.Video" scope="singleton">
        <property name="name" value="tom"/>
        <property name="id" value="23"/>
    </bean>
  • 通过实体类中构造函数注入，必须具有和constructor-arg一致的构造参数，同时记得一定保留无参构造函数
    <bean name="video" class="club.ldclass.spring.domain.Video">
        <constructor-arg name="id" value="1"/>
        <constructor-arg name="name" value="测试constructor-arg"/>
    </bean>
  • POJO类型注入(property 没有使用value属性，而是使用了ref属性)，依赖注入
    <bean name="video" class="club.ldclass.spring.domain.Video" scope="singleton">
        <property name="name" value="tom"/>
        <property name="id" value="23"/>
    </bean>

    <bean name="videoOrder" class="club.ldclass.spring.domain.VideoOrder">
        <property name="id" value="1"/>
        <property name="outTradeNo" value="abcdefg"/>
        <!--依赖于video -->
        <property name="video" ref="video"/>
    </bean>

四、spring ioc容器Bean之间的依赖和继承

  • bean继承：两个类之间大多数的属性都相同，避免重复配置，通过bean标签的parent属性重用已有的Bean元素的配置信息 继承指的是配置信息的复用，和Java类的继承没有关系
//video1实体类
public class Video1 {
    private int id;
    private String name;
    ...
}
//video2实体类
public class Video2 {
    private int id;
    private String name;
    private String summary;
    ...
}
<bean name="video1" class="club.ldclass.spring.domain.Video" scope="singleton">
        <property name="name" value="tom"/>
        <property name="id" value="23"/>
</bean>
<!--通过bean标签的parent属性重用video1的配置信息-->
<bean name="video2" class="club.ldclass.spring.domain.Video2" scope="singleton" parent="video1">
        <property name="summary" value="xxxxxxxx"/>
</bean>
  • 属性依赖: 如果类A是作为类B的属性, 想要类A比类B先实例化，设置两个Bean的依赖关系
    <bean name="video" class="club.ldclass.spring.domain.Video" scope="singleton">
        <property name="name" value="tom"/>
        <property name="id" value="23"/>
    </bean>

    <bean name="videoOrder" class="club.ldclass.spring.domain.VideoOrder" depends-on="video">
        <property name="id" value="1"/>
        <property name="outTradeNo" value="abcdefg"/>
        <!--依赖于video -->
        <property name="video" ref="video"/>
    </bean>

五、bean 的生命周期和二次处理

  • init和destroy方法
//video实体类添加init()和 destroy()
    private void init(){
        System.out.println("video类 init方法被调用");
    }

    private void destroy(){
        System.out.println("video类 destroy方法被调用");
    }
    <bean name="video1" class="club.ldclass.spring.domain.Video" scope="singleton" init-method="init"
          destroy-method="destroy">
        <property name="name" value="tom"/>
        <property name="id" value="23"/>
    </bean>
    //测试
    //控制台打印顺序
    //video类 init方法被调用
    //tom
    //video类 destroy方法被调用
    public static void main(String[] args) {
        ApplicationContext applicationContext = new ClassPathXmlApplicationContext("applicationContext.xml");

        Video video = (Video) applicationContext.getBean("video");
        System.out.println(video.getName());
        //回调函数,触发destroy方法
        ((ClassPathXmlApplicationContext) applicationContext).registerShutdownHook();
    }

六、后置处理器 BeanPostProcessor

  • BeanPostProcessor

    • 是Spring IOC容器给我们提供的一个扩展接口
    • 在调用初始化方法前后对 Bean 进行额外加工，ApplicationContext 会自动扫描实现了BeanPostProcessor的 bean，并注册这些 bean 为后置处理器
    • 是Bean的统一前置后置处理而不是基于某一个bean

  • 执行顺序

    • Spring IOC容器实例化Bean
    • 调用BeanPostProcessor的postProcessBeforeInitialization方法
    • 调用bean实例的初始化方法
    • 调用BeanPostProcessor的postProcessAfterInitialization方法
    • 调用bean实例的销毁方法

  • 自定义CustomBeanPostProcessor

    • 注意：接口重写的两个方法不能返回null，如果返回null那么在后续初始化方法时将报空指针异常或者通过getBean()方法获取不到bean实例对象
public class CustomBeanPostProcessor implements BeanPostProcessor {
    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException {
        System.out.println("CustomBeanPostProcessor postProcessBeforeInitialization beanName="+beanName);
        return bean;
    }
    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException {
        System.out.println("CustomBeanPostProcessor postProcessAfterInitialization beanName="+beanName);
        return bean;
    }
}
  • applicationContext.xml添加bean
    <bean class="club.ldclass.spring.processor.CustomBeanPostProcessor"/>
  • 测试
    public static void main(String[] args) {
        ApplicationContext applicationContext = new ClassPathXmlApplicationContext("applicationContext.xml");
        Video video = (Video) applicationContext.getBean("video");
        //回调函数
        ((ClassPathXmlApplicationContext) applicationContext).registerShutdownHook();
    }
  • 控制台输出
video 注入时被调用
video 注入时被调用
CustomBeanPostProcessor postProcessBeforeInitialization beanName=video
video类 init方法被调用
CustomBeanPostProcessor postProcessAfterInitialization beanName=video
video类 destroy方法被调用
  • 可以注册多个BeanPostProcessor顺序
    • 在Spring机制中可以指定后置处理器调用顺序，通过BeanPostProcessor接口实现类实现Ordered接口getOrder方法，该方法返回整数，默认值为 0，优先级最高，值越大优先级越低
    //CustomBeanPostProcessor.java中添加，表示优先级为1，值越大优先级越低
    public int getOrder(){
        return 1;
    }

七、bean自动装配Autowire属性

  • Spring自动注入

    • 使用元素的 autowire 属性为一个 bean 定义指定自动装配模式
    • autowire设置值
      • no：不开启
      • byName: 根据bean的id/name，注入到对应的属性里面
      • byType：根据bean需要注入的类型，注入到对应的属性里面
        • 如果按照类型注入，存在2个以上相同类型的bean的话会抛异常expected single matching bean but found 2
    • constructor: 通过构造函数注入，需要这个实体类中包含这个类型的构造函数

  • 实例：videoOrder通过三种方式自动注入video

    <bean name="video" class="club.ldclass.spring.domain.Video" scope="singleton">
        <property name="name" value="tom"/>
        <property name="id" value="23"/>
    </bean>

    <bean name="videoOrder" class="club.ldclass.spring.domain.VideoOrder" autowire="byName">
    <!--<bean name="videoOrder" class="club.ldclass.spring.domain.VideoOrder" autowire="byType">-->
    <!--<bean name="videoOrder" class="club.ldclass.spring.domain.VideoOrder" autowire="constructor">-->
        <property name="id" value="1"/>
        <property name="outTradeNo" value="abcdefg"/>
    </bean>
